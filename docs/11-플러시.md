### em.flush()
직접호출하는방법, 자주사용하진않지만 테스트할때 사용할 수 있으므로 알아둬야함  
```
Member member = new Member(200L, "member200");
em.persist(member);

em.flush();
            
System.out.println("===============");
tx.commit();
```
em.flush();를 사용하면 commit가 일어나기전에 insert쿼리를 디비에 날림  
**em.flush();를 사용하면 1차캐시의 내용이 사라지는것이 아니라 쓰기지연SQL저장소에 있는것들이 데이터베이스에 저장됨**

### 트랜잭션 커밋, JPQL 쿼리 실행
플러시자동호출

### 플러시모드 옵션
사용할일이 거이없긴함  
JPQL을 사용할때 플러시가 자동으로 호출되므로 커밋할때 플러시할수있도록 설정하긴함
- FlushModeType.AUTO
  커밋이나 쿼리를 실행할 때 플러시(기본값)
- FlushModeType.COMMIT
  커밋할 때만 플러시
