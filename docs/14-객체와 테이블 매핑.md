# @Entity
@Entity가 붙은 클래스는 JPA가 관리하는 객체라는 뜻입니다. 이 어노테이션을 통해 자바 클래스와 데이터베이스의 테이블이 **매핑(Mapping)** 됩니다.  
- 테이블 매핑: 기본적으로 클래스 이름과 동일한 이름의 테이블과 연결됩니다.
- 인스턴스 관리: JPA는 이 클래스의 인스턴스를 생성하고, 데이터베이스에 저장하거나 조회하는 등 전반적인 생명주기를 관리합니다.

|항목|규칙 및 이유|
|---|--------------------|
|기본 생성자 필수| JPA가 데이터베이스에서 데이터를 조회해 객체를 생성할 때 기술적으로(Reflection) 필요합니다. 접근 제어자는 public 또는 protected여야 합니다.|
|final 클래스 금지| JPA(특히 Hibernate)는 지연 로딩을 위해 엔티티를 상속받은 프록시 객체를 만듭니다. final 클래스는 상속이 불가능하므로 사용할 수 없습니다.|
|final 필드 금지| 데이터베이스의 값을 필드에 채워 넣어야 하는데 final이면 값을 바꿀 수 없기 때문입니다.|
|사용 불가 타입| "enum, interface, inner 클래스는 그 자체를 엔티티로 지정해 사용할 수 없습니다."|
